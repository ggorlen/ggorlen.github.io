Basic examples

Starting out with simple concepts--printing output, introducing variables, conditionals, loops, and functions, and adding style and basic interactivity.

Try it Read it Description
Hello World source The simplest Javascript program that displays output. Is this thing on?
Number guesser source Try to guess the computer's number! An introduction to basic user interaction, conditionals, variables and loops.
Random student selector source Chooses a random item from an array of students.
Google font demonstration source Explore Google's collection of fonts! They're super easy to use and can help make your website or app look snazzy.
Rock, Paper, Scissors source A demonstration of variables, arrays, conditionals, functions, and sending text to the DOM, as well as writing a simple computer opponent--can you think of other ways the computer opponent could behave?

Wordplay

A set of scripts which apply programming rudiments to play with text.

Try it Read it Description
Notepad source A simple note-taking script using jQuery.
Palindrome checker source A palindrome is a word that reads the same forwards and backwards. This is an interesting algorithm that can be written in a variety of ways--this one illustrates using a loop for a purpose a bit more interesting than looking through items in a series.
ROT13 source ROT13 is a simple encryption algorithm for writing secret messages. Each letter in the alphabet is replaced by the letter 13 places ahead of it. This can be accomplished in a few ways--this approach uses an array that contains the entire alphabet to locate the proper rotation letter.
Hangman source Illustrates most programming basics, including loops and array functions. Check the utilities.js file if you'd like to see how to get a random element from an array or read a text file.
Word frequency analyzer source Word frequency analysis builds on ROT13 by using an object to represent a histogram, which counts the number of times a letter shows up in a string. Then, the algorithm uses an array sort to present the output.
Mad libs source Mad libs is a fun game which creates humorous and non-sensical text based on user input, although for simplicity, I just hard-coded in a story with some blanks for the user to fill. This program builds further on arrays and strings, using many language features to perform operations on the data.
Ticker source Ticker is a homebrew news ticker or text scroller that behaves like a blocky version of <marquee>. It demonstrates creating a simple class with a constructor, using string methods and the modulus (%) operator.

Full games

After mastering the basics, here are some full-fledged classic games to check out. Don't worry if the code's a lot to take in--building these games from the ground up yourself is the way to learn! You will find your own way to tackle the challenges a particular game design presents and break the problem down into small components.

Try it Read it Description
Pong source Pong! A classic. It's not difficult to program except for the trickiness of ball physics.
Simple snake source Snake is a game about eating apples and growing longer. I also added poop. It was the first game with graphics I wrote, and I think it's very approachable once you're confident with Javascript basics.
Snake source This version of snake is a little different than simple snake. Sometimes it's nice to write a game a few times. This one adds some animation and uses Javascript's localStorage to keep track of score.
Minesweeper source If you grew up on Windows 3.1 and 95 like I did, you played minesweeper at some point. It's a classic puzzle game, and presents a totally different set of programming challenges from writing an "action" game like snake.
Tetris source Tetris is a game I played on the original Game Boy a ton when I was a kid. When I wrote this version, I very much approached block rotation the wrong way and wrote a ton of code, so I would look at this script as an example of doing something in an intuitive way that isn't actually very good. However, the game works!
Sokoban source Sokoban is a block pushing puzzle game, and I went a bit in-depth adding features on this one. This is a great game for exploring algorithms such as breadth-first search. Even if that's too much for now, keep it in mind for later, or write a simple version that just does the basic game (breadth-first search is used to allow the player to find a path to a mouse click, which isn't mission critical).
